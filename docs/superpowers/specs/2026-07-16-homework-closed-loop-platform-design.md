## 1. 产品概述

本系统是一套家庭私有的在线作业闭环平台。家长统一录入作业本体并邀请分科家教；孩子只负责查看任务、开始、暂停、完成及填写不会的题号；家教负责本科计划调整、批改、订正验收、知识点评级和学校平台提交确认。

系统不代替学校 PAD 或其他学校平台提交作业，不强制保存孩子作业照片。系统的价值是让每项作业都能从计划、独立作答、批改、订正、复做、知识掌握到学校提交形成可追溯闭环。

日历只是任务的一种视图，数据核心是：

`作业任务 + 知识点 + 执行计划 + 学习证据 + 状态机 + 提交确认`

## 2. 背景与问题

### 2.1 当前问题

1. 暑假作业分散在 PDF、Word、家长会口头要求和学校平台中。
2. 不同科目的答案发布和提交节奏不同，不能用统一“完成”复选框表示。
3. 家长难以同时追踪多名分科家教的计划调整、批改和提交情况。
4. “孩子做过”“家教批改过”“孩子订正过”“学校平台已提交”经常被混为一体。
5. 作业完成记录没有沉淀为知识点掌握证据，无法形成长期能力曲线。
6. 现有课表和学校提交计划会变化，需要允许家教直接调整本科计划并保留记录。

### 2.2 当前作业规模

- 数学：15 个文件，实际 19 个子卷，共 351 道编号题。
- 物理：26 份作业，共 383 道编号题。
- 生物：9 套测试，共 225 道编号题。
- 语文：8 套综合卷，共 184 道编号题，另有《红楼梦》阅读与思考题。
- 化学：正式必刷题 1—17，另有预习和电化学检测材料。
- 俄语：30 次每日一练，作业正文尚未导入。

## 3. 产品目标

### 3.1 第一阶段目标

1. 完整导入当前暑假作业、学校截止时间和知识点体系。
2. 让分科家教一登录就知道本科今天和本周应处理什么。
3. 让家教以勾选为主完成批改、订正验收、掌握确认和学校提交确认。
4. 让家长无需逐科询问即可看到全科进度、风险和变更。
5. 让孩子只能操作与独立学习直接相关的最少功能。
6. 防止“做过但未批改”“批改但未订正”“订正但未提交”被误判为完成。
7. 把每次练习沉淀为知识点证据，为长期学习平台提供可靠数据。

### 3.2 长期目标

- 管理日常作业与阶段考试。
- 建立知识点掌握变化、错误类型和完成效率曲线。
- 将考试结果与日常练习证据关联。
- 在证据充分后提供练习建议，而非用一次正确率给孩子贴标签。

## 4. 第一阶段非目标

第一版明确不包含：

- 自动连接学校 PAD 或代交作业。
- 微信、短信、邮件、浏览器外部推送。
- 强制作业照片存储。
- AI 自动识别 PDF 或照片并生成任务。
- 家教在线聊天。
- 逐题复杂批注工具。
- 考试管理、成绩预测和智能推荐练习。
- 英语、政治、历史、地理作业。

## 5. 已确认的排程约束

1. 删除 7 月 5 日全部安排。
2. 语文任务归入“考背”类别。
3. 语文前四次学校截止早于现有考背课，系统初始计划必须增加考背自主任务块。
4. 第七、八套语文卷必须提前启动，不能只依靠 8 月 24—28 日三个考背任务块。
5. 计划按日期管理，不要求填写具体开始时刻。
6. 每个执行块统一显示为 90 分钟。
7. 学校正式截止仍保存精确日期和时间，例如当日 21:00 或次日 8:00。
8. 每日建议容量默认 2 个任务块，家长可按实际课程负荷调整。

## 6. 用户与权限

### 6.1 家长管理员

家长拥有家庭空间的管理权：

- 创建和维护孩子档案。
- 邀请、移除家教并按科目授权。
- 创建、修改、归档作业本体。
- 维护学校截止时间和提交节点。
- 关联知识点和必做/选做属性。
- 查看全部科目、计划、批改、知识和审计记录。
- 导出完整学习档案。

家长不需要逐项批准家教的本科计划变更，只接收站内变更通知。

### 6.2 分科家教

家教只能访问被授权的孩子和科目：

- 查看本科作业本体和知识点。
- 修改本科执行计划日期。
- 拆分、合并尚未开始的 90 分钟执行块。
- 增加续做、批改、订正、复做任务块。
- 完成勾选式批改和掌握评价。
- 确认学校平台提交节点。
- 查看本科知识证据、风险和变更历史。

家教不能修改学校布置的作业内容、删除历史学习证据或查看其他家教的科目。

### 6.3 孩子

孩子可以：

- 查看自己的全部科目和今日/本周任务。
- 点击开始、暂停、继续和我已完成。
- 填写不会的题号。
- 查看精简的知识点点亮结果。

孩子不能：

- 修改计划和截止时间。
- 查看答案、批改记录编辑页或提交确认页。
- 修改知识点评级。
- 上传作业或确认学校提交。

### 6.4 系统

系统负责：

- 状态流转和完成条件判断。
- 截止风险和每日负荷计算。
- 计划、批改、提交与权限的审计。
- 站内通知和周报生成。
- 知识点点亮、回落和证据汇总。

## 8. 作业本体与计划分离

### 8.1 作业本体

作业本体表达学校布置了什么，由家长维护。内容包括：

- 科目、批次、标题、材料页码和题号。
- 学校截止时间。
- 必做、选做或材料状态。
- 对应知识点。
- 答案状态：未发布、已发布、附件内已有。
- 所需提交节点。

孩子开始后，家长再修改作业内容时必须创建新版本，旧学习记录继续关联旧版本。

### 8.2 执行计划

执行计划表达何时做、分几次做，由分科家教维护：

- 每个任务块统一 90 分钟。
- 只排日期，不排具体钟点。
- 一个作业可对应多个首做、续做、批改、订正、复做任务块。
- 多个任务块仍只归属于一项作业，不能重复计算作业完成数。

### 8.3 任务块类型

- 知识回顾
- 独立首做
- 续做
- 家教批改
- 孩子订正
- 独立复做
- 提交确认
- 阅读/背诵

## 9. 双轨状态机

### 9.1 作业流程轨

`待安排 → 待开始 → 进行中 → 孩子已完成 → 待批改 → 待订正 → 待复做 → 待验收 → 已闭环`

规则：

1. 孩子点击我已完成后进入待批改。
2. 家教批改无错误时可直接进入待掌握确认。
3. 存在必改错误时必须进入待订正。
4. 家教可要求一轮或多轮订正。
5. 需要独立复做的知识点，复做通过后才能标为已掌握。
6. 已闭环任务可由家教重新打开，操作必须进入审计记录。

### 9.2 学校提交轨

一项作业可配置多个提交节点：

- 首次提交
- 订正后回传
- 纸质作业保留
- 自定义节点

每个节点包含：未到节点、待确认、已确认、已撤销四种状态。

“已批改”和“已在学校平台提交”必须分别确认。确认时自动记录家教和时间；截图、备注选填。

### 9.3 完整闭环条件

作业只有同时满足下列条件才显示为闭环完成：

- 孩子完成首做。
- 家教完成批改。
- 所有必改错题订正通过。
- 被要求的独立复做通过。
- 家教已经确认掌握等级。
- 所有必需提交节点均已确认。

学校提交确认不直接改变知识掌握等级。

## 10. 知识点体系与点亮规则

### 10.1 层级

`科目 → 模块 → 单元 → 知识点 → 能力标签`

示例：

`数学 → 平面向量 → 向量运算 → 数量积 → 夹角与投影应用`

### 10.2 作业关联

一项作业可关联多个知识点，每个关联可标记：

- 主要知识点
- 次要知识点
- 综合应用
- 必须掌握
- 拓展选做

家教在批改时可勾选“暴露了其他知识点问题”，生成诊断证据，但不能修改家长维护的原始映射。

### 10.3 点亮状态

| 等级 | 颜色 | 触发条件 |
|---|---|---|
| 未练习 | 灰 | 没有有效练习证据 |
| 已练习 | 蓝 | 孩子完成首做 |
| 有待巩固 | 橙 | 存在错题或订正未通过 |
| 基本掌握 | 浅绿 | 订正通过但未完成独立复做 |
| 已掌握 | 绿 | 订正和独立复做均通过 |

点亮不是永久锁定。后续新练习再次暴露问题时，最近状态可回落，同时保留历史最高等级。

### 10.4 证据链

知识点详情页展示：

- 关联作业和题号。
- 首做正确率区间。
- 错题号和错误类型。
- 订正轮次与结果。
- 独立复做结果。
- 家教最近评价。
- 对应学校提交状态。

## 11. 勾选式批改

### 11.1 必填步骤

家教完成一次批改最多需要五组必填操作：

1. 正确率区间：100%、90%以上、70%—89%、70%以下。
2. 错题号：支持快速输入，例如 `3、7、12(2)`。
3. 错误类型多选。
4. 订正是否通过。
5. 独立复做是否通过及掌握等级。

### 11.2 错误类型

- 概念不清
- 公式或记忆错误
- 审题错误
- 建模错误
- 计算错误
- 步骤不规范
- 粗心
- 时间不足

评语、备注和截图全部选填。

## 12. 计划变更与风险

### 12.1 家教可直接变更

分科家教无需审批即可：

- 移动本科任务块日期。
- 拆分或合并尚未开始的任务块。
- 增加续做、订正和复做块。
- 移动单个块或整条后续链路。

家长收到站内变更通知。

### 12.2 变更记录

记录内容：

- 修改人和修改时间。
- 原日期和新日期。
- 修改类型。
- 是否影响后续链路。
- 原因：课程冲突、孩子未完成、临时请假、难度超预期、学校截止变化、其他。

### 12.3 风险检查

系统检查：

- 当日任务块是否超过家庭设定容量。
- 剩余任务块是否能在学校截止前完成。
- 是否为批改、订正和提交预留任务块。
- 是否有任务块安排在学校截止之后。
- 是否存在跨科目日期负荷冲突。

家教可在红色风险下强制保存，但必须选择原因。

## 13. 页面与信息架构

### 13.1 视觉原则

- iOS 风格。
- 大标题、圆角卡片、分段控制器、底部标签栏。
- 文字精简，状态和下一步操作优先。
- 手机端单手可完成主要操作。
- 颜色必须配合文字或图标，不能只靠颜色传递信息。

### 13.2 家长端

底部导航：`总览 / 日历 / 作业 / 知识 / 设置`

首页内容：

- 本周任务块、截止风险、知识点亮和提交概况。
- 需要关注的作业。
- 家教最近计划变更。
- 录入新作业入口。

### 13.3 家教端

底部导航：`今天 / 日历 / 批改 / 知识 / 我的`

家教首页采用“日历优先”：

- 周/月/变更三个视图。
- 按日期显示任务块，不显示具体钟点。
- 每个任务块显示 90 分钟、状态、学校截止和主操作。
- 日负荷过高以警示点和风险卡显示。

### 13.4 孩子端

底部导航：`今天 / 本周 / 知识 / 我的`

首页内容：

- 今日任务块和总分钟数。
- 当前任务的开始、暂停、继续、完成。
- 不会题号入口。
- 精简知识点点亮和下一项任务。

## 14. 看板与周报

### 14.1 家长总览指标

- 计划任务块数和完成数。
- 待批改、待订正、待提交数量。
- 即将截止和已经逾期的作业。
- 各科知识点点亮比例。
- 最近家教计划变更。
- 各科批改和提交确认状态。

### 14.2 家教看板指标

- 今日与本周任务块。
- 待批改、待订正验收、待提交。
- 本周高频错误类型。
- 橙色知识点和待复做知识点。
- 本科截止风险和变更历史。

### 14.3 孩子反馈

- 今日剩余任务块。
- 连续完成天数。
- 本周新点亮知识点。
- 需要再练一次的知识点。

孩子端不展示家教效率、逾期排名或复杂统计。

### 14.4 自动周报

- 计划完成率。
- 学校提交确认率。
- 首做正确率区间。
- 订正完成率。
- 独立复做通过率。
- 新点亮、回落、待巩固知识点。
- 下周优先处理事项。

周报只保存在系统内。

## 15. 站内通知

通知类型：

- 新任务分配。
- 计划变更。
- 学校截止变化。
- 截止风险。
- 孩子完成、待家教批改。
- 订正完成、待家教验收。
- 提交节点待确认。
- 家教邀请、授权和移除。

系统不发送微信、短信、邮件或浏览器外部通知。

## 16. 异常与数据一致性

1. 家长在孩子开始后修改作业：创建新版本，不覆盖旧记录。
2. 学校截止变化：重新计算风险并通知相关家教。
3. 家教移动计划：保留原计划和新计划，不清空学习进度。
4. 误点已批改或已提交：允许撤销，但必须选择原因。
5. 重复确认：操作幂等，只产生一条有效记录。
6. 网络中断：未提交的勾选内容在本地恢复，联网后继续。
7. 家教被移除：未完成任务回到待分配，历史批改保留署名。
8. 多人同时修改：后保存者看到冲突，不静默覆盖。
9. 知识点映射改变：旧证据保留旧版本，新练习使用新映射。
10. 删除使用软删除；作业、计划、批改和提交记录可恢复。

## 17. 安全与隐私要求

- 私有家庭空间，默认不公开分享。
- 家教采用按孩子、按科目的最小权限。
- 所有权限在服务端校验。
- 邀请链接限时且仅可使用一次。
- 移除家教后访问权限立即失效。
- 敏感操作进入不可无痕删除的审计记录。
- 默认不保存作业照片。
- 支持家长导出和备份完整数据。
- 数据时间以 Asia/Shanghai 为默认显示时区。

## 18. 非功能要求

### 18.1 性能

- 常用页面在正常网络下 2 秒内可操作。
- 列表分页或增量加载，长期数据增长后不一次加载全部记录。
- 批改勾选自动保存，避免页面退出丢失。

### 18.2 兼容性

- 支持手机和桌面浏览器。
- 手机宽度 390 像素时主要功能完整可用。
- 桌面端支持批量浏览、筛选和录入。

### 18.3 可用性

- 主要操作按钮使用动词：开始、暂停、完成、批改、验收、确认提交。
- 不使用含义模糊的单独“完成”状态。
- 危险操作二次确认。
- 状态颜色同时配有文字或图标。

### 18.4 可靠性

- 关键写操作支持幂等。
- 定期自动备份。
- 软删除可恢复。
- 计划、批改和提交确认具有完整审计链。

## 20. 验收标准

1. 家教无法访问未授权科目。
2. 孩子无法修改计划、批改或提交状态。
3. 家教可在不写长评语的情况下完成批改。
4. 一次批改必填操作不超过五组。
5. 计划调整后历史记录和已完成进度不丢失。
6. 一个作业可拆为多个 90 分钟块，但作业完成数只统计一次。
7. 学校提交确认不自动点亮知识掌握。
8. 订正和必要复做通过后，知识点才能显示绿色。
9. 撤销批改或提交确认必须留下原因和时间。
10. 家长可查看全部科目风险和家教变更。
11. 手机端可单手完成主要操作，桌面端可高效查看。
12. 所有颜色状态同时配有文字或图标。
13. 常用页面正常网络下 2 秒内可操作。
14. 网络中断后未提交的批改内容可以恢复。
15. 当前暑假作业、截止时间和知识点能够完整导入。
16. 7 月 5 日不生成任何初始计划块。
17. 语文任务显示在考背类别，并为前四次截止增加自主任务块。
18. 所有必需提交节点确认后，提交轨才显示完成。

## 21. 测试重点

### 21.1 权限测试

- 家教跨科访问必须被服务端拒绝。
- 孩子访问批改、答案和提交确认接口必须被拒绝。
- 移除家教后旧登录会话立即失去相关权限。

### 21.2 状态机测试

- 正常首做、批改、订正、复做和提交链路。
- 无错题直接通过链路。
- 多轮订正链路。
- 首次提交后开放答案、订正后回传链路。
- 撤销和重新打开链路。

### 21.3 计划测试

- 移动单个任务块和整条链路。
- 拆分/合并任务块。
- 截止前容量不足提示。
- 计划在截止后保存时的红色风险和强制原因。
- 家长变更学校截止后的重新计算。

### 21.4 数据一致性测试

- 重复点击不重复生成确认记录。
- 并发修改不静默覆盖。
- 作业版本变化不污染历史证据。
- 知识映射变化保留旧证据。
- 软删除和恢复。

### 21.5 端到端测试

从家长创建任务开始，完整验证：

`授权家教 → 生成计划块 → 孩子完成 → 家教批改 → 孩子订正 → 家教验收 → 确认学校提交 → 知识点点亮 → 周报更新`

## 22. 上线成功标准

- 不漏掉任何学校正式截止。
- 每项作业可追溯到负责家教、执行块和知识点。
- 家教登录后可立即看到本科待办和风险。
- 家长无需逐科询问即可了解进度。
- 中间状态不会被错误标为完成。
- 暑假结束后可导出完整作业、批改、提交和知识档案。

## 24. 数据导入说明

第一批导入范围：语文、数学、俄语、物理、化学、生物。

- 英语、政治、历史、地理不导入。
- 俄语正文未提供，先创建 30 个每日练习任务壳；取得材料后再关联知识点。
- 化学 PDF 第 45—62 页与正式计划不完全一致，导入为“拓展/材料状态”，不设为强制提交，直到家长依据学校通知调整。
- 生物额外知识点填空和三套提升卷未提供，不在第一批任务中。
- 语文征文题目和截止未提供，不生成正式提交节点。

## 25. 设计结论

第一版应实现一个轻量但严格的闭环工作流：家长负责作业本体，家教负责本科执行和确认，孩子只负责独立学习动作。系统以日期级 90 分钟任务块承载计划，以双轨状态机区分学习与提交，以知识证据链决定点亮，以审计记录保证计划变化可追溯。

此设计既能在暑假阶段解决实际提交问题，又不会把第一版扩张成复杂学习 ERP；同时保留未来接入日常作业、考试和长期能力曲线所需的数据基础。

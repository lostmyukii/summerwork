## 1. 实施原则

1. 预习是独立业务实体，不复用学校作业任务，不进入作业完成率、批改率、掌握率或学校提交率。
2. 23 节预习全部占用已确认的家教新课 90 分钟，不额外增加当天总时长；8 月 12 日不执行，M03 顺延到 8 月 14 日。
3. 家教为预习主操作人；家长和孩子只读。预习状态固定为“待带学 → 已带学 → 验收通过”。
4. 家教只录入实际完成题数，勾选预设未掌握知识点，并可补充自定义未掌握点。
5. 预习状态、作业流程、知识掌握和学校提交互不代替；日历负责展示关系，不承担数据权威。
6. 每批先写测试、再实现最小纵切；代码用 Git 检查点恢复，数据库用迁移前备份和项目内补偿 SQL 恢复。

## 2. 批次 A：结构化预习计划与规则

目标文件：

- `app/lib/prestudy-plan.ts`
- `tests/prestudy-plan.test.ts`
- `scripts/sync-prestudy-plan.mjs`

任务：

1. 从工作簿沉淀 23 节预习的稳定编号、原日期、执行日期、科目、主题、四阶段带学内容、验收标准和预设知识点。
2. 固化 8 月 12 日不执行、M03 在 8 月 14 日执行，以及语文仅在 8 月考背家教时段执行。
3. 校验每节 90 分钟、六科数量、日期和知识点完整性；原工作簿继续留在仓库外。
4. 提供幂等同步脚本，按家庭、孩子、科目家教分配同步，不触碰学校作业表。

验收：23 个稳定编号唯一；科目数量为数学 5、物理 5、化学 4、生物 4、俄语 3、语文 2；M03 只在 8 月 14 日执行。

## 3. 批次 B：数据库、权限与命令

目标文件：

- `supabase/migrations/0021_independent_prestudy_track.sql`
- `tests/supabase-schema.test.ts`
- `tests/supabase-workflow.integration.sql`

任务：

1. 新增预习课、预设知识点、执行记录和未掌握知识点表，并建立唯一键、版本号、时间戳和必要索引。
2. 家长和孩子可读取本人家庭全部预习；家教只读取和操作有效分科授权范围内的预习。
3. 用受控命令实现“已带学”“保存验收”“撤销状态”和“调整本科日期”，拒绝跨科、越级和过期版本写入。
4. 验收必须发生在带学之后，并要求实际完成题数为大于等于 0 的整数；验收不自动点亮知识掌握。
5. 所有状态与日期变更写入审计；通知仅保存在系统内。

验收：重复命令幂等，家长和孩子不能写，跨科家教不能读写，预习操作不改变任何学校作业统计。

- `app/lib/supabase/prestudy-actions.ts`

2. 服务端并行加载预习数据，避免与现有大任务查询形成串行瀑布。

4. 预习日期变更后刷新工作台和内部通知，保留原计划日期。

验收：真实账号按角色获得正确数据；无预习家庭仍能正常打开现有作业工作台。

## 5. 批次 D：上下双轨界面

目标文件：

- `app/components/prestudy-track.tsx`
- `app/components/prestudy-workspace.tsx`
- `app/components/homework-platform.tsx`
- `app/globals.css`
- `tests/rendered-html.test.mjs`

任务：

1. 日历日期内固定显示上方“预习线”、下方“作业线”，使用文字标签和状态，不只依赖颜色。
2. 家教工作台提供“预习｜作业”分段切换；预习页集中完成带学、实际题数、未掌握项和验收。
3. 家长、孩子可以查看预习主题、带学内容、状态、实际题数和未掌握项，但没有编辑控件。
4. 8 月 14 日数学 M03 与物理 P03 同日分科显示；8 月语文预习归入考背家教身份。
5. 保持现有 iOS 风格，预习使用冷蓝色轨道签，作业沿用原暖色；手机 390px 下不横向溢出。
6. 长列表使用分组和可见性优化，派生索引使用 `Map`，避免在大组件中反复扫描 203 个作业任务。

验收：三角色能一眼区分两条线；完成预习不改变作业卡状态；键盘焦点、移动端和空状态可用。

## 6. 批次 E：本地验收与发布

1. 运行 `npm run typecheck`、`npm run lint`、`npm run test:unit`、`npm run build` 和 `npm run test:render`。
2. 本地 PostgreSQL 可用时运行 `npm run test:db:local`，覆盖家长、分科家教、孩子和跨科拒绝路径。
3. 浏览器检查桌面与 390px 手机的日历双轨、家教预习操作、家长和孩子只读状态。
4. 发布前记录 Git 检查点，检查工作区只包含本项目变更；不提交 `.superpowers/` 和用户现有的未跟踪文件。
5. 服务器先检查 `/srv/summerwork`、容器、端口、网络、卷和其他站点健康，再备份 Summerwork 数据库与当前应用版本。
6. 仅应用 `0021` 迁移，幂等同步 23 节预习，先用合成家长、分科家教、孩子验收，再部署应用。
7. 运行 `nginx -t`、两域名健康检查和其他既有项目抽查；失败时只回退 Summerwork 应用版本，并用备份或项目内补偿脚本恢复本次数据库变更。

发布验收：线上三角色可读取正确预习；家教可完成带学与验收；家长、孩子只读；学校作业 203 条及历史状态数量不变；其他项目无健康变化。

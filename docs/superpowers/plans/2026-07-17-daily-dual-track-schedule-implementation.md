# 逐日双轨清单入系统实施计划

## 目标

把已确认的逐日清单变成系统权威数据，而不是静态文档：43节数理化生预习、203项学校作业、18个旅行自主任务全部可在家长/家教/孩子日历中按日期和科目查看，并保留每项作业原有的首做、批改、订正、复做、掌握和学校提交状态。

## 不变量

- 预习线只覆盖数学、物理、化学、生物；俄语、语文不显示预习线。
- 2026-07-26 至 2026-08-12 是唯一允许自主作业的区间，每天最多1项，共18项。
- 旅行期外全部作业归入家教课内；容量不足显示“本科课内积压”，不转成普通自习。
- 2026-08-16、2026-08-23 无执行任务，只保留截止和提交提示。
- 语文7月占生物课45分钟，8月进入考背课90分钟；生物8月保留完整90分钟。
- 203项任务只迁移计划日期和课内归属，原始日期、工作流、知识点和提交记录不丢失。

## 数据实现

1. 新增结构化清单 `app/data/daily-dual-track-2026.json`，记录81个作业块及其203项任务映射、容量、初估用时、积压分钟、旅行补位日期。
2. 将预习源扩展为43节，并为预习课增加 `active` 状态；同步时停用俄语/语文旧预习，不删除历史执行记录。
3. 新增 `study_blocks` 与 `study_block_items`，用独立块保存课内队列和旅行自主队列；任务工作流仍以原 `homework_tasks` 为核心。
4. 同步脚本逐项更新 `planned_date`、`slot_type`、`course_integrated`，为日期变化写入 `task_plan_changes` 与 `change_events`，确保家长可追溯。

## 界面实现

- 日历继续使用上下双轨：上方预习线，下方学校作业线。
- 学校作业按“日期＋科目＋作业块”分组，块头展示课内容量、任务数、初估分钟和积压。
- 旅行自主块显示补位家教日期；课内积压只显示在本科块内。
- 俄语/语文家教工作台默认进入作业线，不显示空的预习操作入口。
- 每项作业仍保留独立的进入批改、调整计划、提交确认和知识闭环操作。

## 验收与发布

1. 本地校验：203项任务恰好映射一次、43节预习、18个旅行日每天1项、旅行外自主0项、8月16/23执行任务0项。
2. 运行类型检查、Lint、单元测试、构建和渲染测试。
3. 发布前记录 Git 检查点；在线上仅备份 `/srv/summerwork` 数据库、compose配置和当前应用镜像。
4. 应用迁移与同步后，用家长、数学家教、俄语家教、孩子合成账号验证可见范围和上下双轨。
5. 对比部署前后同机其他容器、端口与Nginx配置；异常时只回滚 `summerwork` 数据库和应用镜像。
